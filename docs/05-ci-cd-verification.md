# CI/CD Verification

This step validates the CI/CD pipeline after initial setup.


## 🔁 Process

1. Modify the application source code  
2. Push changes to GitHub (if working locally)  
3. Jenkins automatically triggers the pipeline  
4. Build is executed and deployed to Tomcat  
5. Verify updated changes on the deployed application  


## ✅ Outcome

- Confirms end-to-end CI/CD automation is working
- Ensures continuous deployment is functional
